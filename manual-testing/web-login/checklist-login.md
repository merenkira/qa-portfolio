# Checklist – Login Page (Web)

This checklist covers basic functional and UI checks for a web login form.

- [ ] Login page loads without errors (status 200, no console errors)
- [ ] Email and password fields are visible and enabled
- [ ] Login button is visible and enabled by default (according to requirements)
- [ ] Email field validates incorrect format (e.g. `user@`, `user@domain`)
- [ ] Required field validation is shown when email is empty
- [ ] Required field validation is shown when password is empty
- [ ] Correct error message is shown for invalid credentials
- [ ] Error message disappears after a successful login
- [ ] “Remember me” option works according to requirements
- [ ] User is redirected to the correct page after successful login
- [ ] User cannot access protected pages without being logged in
- [ ] Password field hides characters by default
- [ ] Login form behaves correctly on different screen sizes (responsive)
- [ ] Basic accessibility: labels are linked to inputs, focus order is logical
